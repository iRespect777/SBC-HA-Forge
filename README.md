# 🏠 Home Assistant Supervised - Ultimate Installer for TV-Boxes & SBC

[![Bash](https://img.shields.io/badge/Language-Bash%20%3E%3D%204.0-green.svg)](https://www.gnu.org/software/bash/) [![License: MIT](https://img.shields.io/badge/License-MIT-blue.svg)](https://opensource.org/licenses/MIT)
 
**Platform:** Armbian / Debian (Bookworm, Bullseye, Trixie) | aarch64 / x86_64 / armv7  

Этот скрипт превращает любую ТВ-приставку (X96, Tanix, HK1 на Amlogic/Allwinner), одноплатный компьютер (Raspberry Pi, Odroid) или мини-ПК в полноценный, защищенный и отказоустойчивый сервер **Home Assistant Supervised**.

Установка HA Supervised на нестандартное железо сопряжена с массой проблем: от нестабильных Wi-Fi и отваливающегося VPN до необходимости патчить загрузчик для AppArmor на системах с двумя дисками (eMMC + SD). Этот скрипт решает все проблемы автоматически, предоставляя интеллектуального мастера установки и набор утилит для пожизненного обслуживания сервера.

---

## ✨ Ключевые особенности

- 🧠 **Умный поиск загрузчика (SD/eMMC):** Автоматически находит конфиги загрузчика на отмонтированных разделах. Если найдено два диска (например, загрузчик на SD, а система на eMMC), скрипт спросит, какой именно патчить, и безопасно пропишет его в `/etc/fstab` по UUID.
- 🛡️ **Отказоустойчивость (Resilience):** Автоматический откат сети (Rollback) при обрыве SSH во время настройки NetworkManager.
- 🔄 **Продолжение после перезагрузки:** Если AppArmor требует перезагрузки, скрипт создает systemd-сервис, который **автоматически продолжит установку** с прерванного шага после рестарта.
- 🕵️‍♂️ **Подмена `os-release`:** Обманывает HA Supervisor, динамически подменяя `/etc/os-release` на Debian, позволяя HA работать на Armbian/Ubuntu.
- 📡 **Удаленный доступ из коробки:** Глубокая интеграция с **Tailscale** (частная сеть) и **Cloudflare Tunnel** (публичный HTTPS без портов).
- 📶 **Фикс обрывов Wi-Fi:** Автоматически отключает агрессивное энергосбережение (Power Save) для Wi-Fi адаптеров через NetworkManager (по UUID соединения), спасая VPN от разрывов.
- 📦 **Умные бэкапы:** Динамическое определение пути хранения снапшотов HA (поддержка Debian-путей и Docker Volumes). Локальные и удаленные бэкапы (SSH/Rclone).
- 🔒 **Безопасность:** Настройка UFW (включая правила `DOCKER-USER`), Fail2Ban, защита SSH.
- 🐢 **Оптимизация для eMMC/SD:** Настройка ZRAM, профиля CPU Governor, ограничения логов, опции `noatime/commit=600` в fstab.

   - TV-Box или SBC.

   - Чистая установка Armbian (Debian-based: Bookworm, Bullseye, Trixie).
3. **Доступ:**
   - Root-доступ (`sudo`).
   - Стабильное интернет-соединение.

## 🔬 Глубокое погружение (Как это работает)

### 1. Поиск раздела загрузчика (Boot Directory) и AppArmor
На TV-приставках раздел с загрузчиком часто не монтируется автоматически, или система работает с SD-карты, а корень `/` находится на eMMC. 
Скрипт действует так:
1. Сканирует стандартные точки монтирования.
2. Если не находит — ищет отмонтированные партиции с метками `BOOT` или `armbian`.
3. **Если найдено два диска (SD + eMMC)** — выводит интерактивное меню с подсказками (на каком диске лежит корень системы), чтобы пропатчить именно тот, с которого происходит загрузка.
4. Примонтированный раздел добавляется в `/etc/fstab` **строго по UUID** (безопасно для ARM), чтобы параметры AppArmor применялись при каждой загрузке.

### 2. Удаленный доступ: Tailscale vs Cloudflare Tunnel
Скрипт предлагает два разных подхода к удаленному доступу, которые можно использовать одновременно:

*   **Tailscale (Частная сеть):** Идеально для безопасного доступа к SSH и HA со своего телефона/ПК. Работает как расширение локальной сети (WireGuard). Скрипт автоматически открывает порт `41641/udp` в UFW для прямых P2P-соединений.
*   **Cloudflare Tunnel (Публичный HTTPS):** Идеально для интеграции с Яндекс.Алисой, Telegram Webhooks или доступа с работы. Не требует DDNS, статического IP или открытия портов. Скрипт скачивает бинарник `cloudflared` и регистрирует systemd-сервис по токену из Zero Trust Dashboard.

### 3. Фикс отваливающегося Wi-Fi и VPN
Адаптеры Wi-Fi на TV-боксах агрессивно засыпают, обрывая Tailscale. Установщик использует `nmcli` для надежного отключения `802-11-wireless.powersave`, работая с подключениями через **UUID** (а не имена SSID), что исключает ошибки, если в имени сети есть пробелы или кириллица.

### 4. Динамический поиск бэкапов (ha-backup-remote)
В зависимости от способа установки (Debian-пакет или Docker/OVA), Home Assistant сохраняет снапшоты в разных местах (`/var/lib/homeassistant/backup`, `/usr/share/hassio/backup` или внутри Docker Volumes). Скрипт `ha-backup-remote` динамически определяет путь к папке, прежде чем загружать архив в облако (rclone/SSH).

## 🛠 Пост-установочные утилиты

Скрипт разворачивает набор утилит в `/usr/local/bin/` и настраивает для них `cron`:

| Утилита | Описание | Расписание |
| :--- | :--- | :--- |
| `ha-health` | Вывод сводки о здоровье системы (RAM, Disk, контейнеры). | Вручную |
| `ha-backup` | Создание бэкапа. Автоматически использует HA CLI для полных снапшотов. | Каждое воскресенье 04:00 |
| `ha-restore` | Интерактивное восстановление из бэкапа. | Вручную |
| `ha-backup-remote` | Умная отправка последнего бэкапа в облако (rclone/SSH). | Каждое воскресенье 04:30 |
| `ha-notify "Текст"` | Отправка уведомления в Telegram или Webhook. | Вручную |
| `ha-watchdog` | Следит за HA. При падении перезапускает с экспоненциальной задержкой. | Каждые 5 минут |
| `ha-cleanup` | Очищает Docker кэш и логи, если на диске меньше 1.5 ГБ. | Ежедневно 03:30 |
| `ha-net-recovery` | Пингует шлюз. Если сети нет, перезапускает NetworkManager. | Каждые 10 минут |

## 🚑 Решение проблем

### Tailscale отваливается через несколько минут
Это признак того, что ваш Wi-Fi адаптер ушел в спящий режим (Power Save). Установщик пытается отключить это, но если что-то пошло не так:
1. Узнайте UUID вашего Wi-Fi: `nmcli con show`
2. Отключите энергосбережение: `sudo nmcli con modify "ИМЯ_СЕТИ" 802-11-wireless.powersave 2`
3. Переподключитесь: `sudo nmcli con up "ИМЯ_СЕТИ"`

### Cloudflare Tunnel не запускается
Убедитесь, что вы сгенерировали токен в Cloudflare Zero Trust Dashboard -> Networks -> Tunnels, и установили сервис командой:
```bash
sudo cloudflared service install <ВАШ_ТОКЕН>
```

### Не работает DNS внутри контейнеров (ошибка загрузки HACS)
Установщик добавляет `"dns": ["8.8.8.8", "1.1.1.1"]` в `/etc/docker/daemon.json`. Если HACS всё равно не скачивается:
```bash
docker restart homeassistant
```

### Скрипт завис или оборвался по SSH
Не паникуйте! Скрипт использует систему состояний (`/var/lib/ha-installer/state`). Просто переподключитесь по SSH и запустите скрипт снова. Он продолжит установку с прерванного шага.

## 📂 Структура файлов установщика

- `/var/lib/ha-installer/` — Рабочий каталог установщика.
  - `state` — Файл состояния (какие шаги выполнены).
  - `config` — Конфигурация установки (профиль, версии, пути).
  - `secrets/` — Защищенные токены (Telegram, Tailscale, Cloudflare, Webhooks).
  - `backup/` — Бэкапы системных файлов (os-release, interfaces, fstab, конфиги загрузчика).
- `/var/backups/homeassistant/` — Каталог для локальных бэкапов HA (tar.gz).
- `/usr/local/bin/ha-*` — Утилиты управления (watchdog, backup, notify).
- `/var/log/ha_install_*.log` — Подробные логи последней установки.
