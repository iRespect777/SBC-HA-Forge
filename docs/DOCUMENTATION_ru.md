# 📖 Полная документация Hassify

**Hassify** — это сложный оркестратор для автоматизации установки Home Assistant Supervised на Single Board Computers (SBC) и TV-боксы. Скрипт берет на себя всю подготовку хост-системы, обход аппаратных ограничений, настройку безопасности и резервного копирования.

## 📋 Требования к системе

Home Assistant Supervised имеет строгие требования к хост-системе. Скрипт проверяет их на этапе предустановки (Preflight).

*   **Архитектура:** `x86_64` (Intel/AMD) или `aarch64` (ARM 64-bit). *(Установка на 32-битные armv7l принудительно заблокирована, так как HA более не выпускает для них пакеты).*
*   **Операционная система:** Debian 12 (Bookworm) или Debian 13 (Trixie). Рекомендуется [Armbian](https://www.armbian.com/).
*   **Ядро Linux:** Обязательно наличие **cgroups v2**. Установка на ядра с cgroups v1 прервется с ошибкой.
*   **ОЗУ (RAM):** Минимум 1 ГБ (рекомендуется 2 ГБ+).
*   **Накопитель:** Минимум 4 ГБ свободного места (рекомендуется 10 ГБ+).
*   **Права:** Запуск только от `root` (через `sudo`).

## 🚀 Установка и Быстрый старт

Выполните одну команду в терминале вашего устройства:

```bash
wget https://raw.githubusercontent.com/iRespect777/Hassify/refs/heads/main/ha-installer/Hassify.sh
sudo bash Hassify.sh
```

После завершения установки скрипт скопирует себя в `/usr/local/bin/ha-install` и будет доступен для вызова из любой директории.

### Защита от обрыва SSH
Если во время установки (например, при переключении сети на NetworkManager или во время перезагрузки для AppArmor) SSH-сессия оборвется, скрипт продолжит работу в фоне через службу systemd. 
Чтобы увидеть процесс:
1. Зайдите по SSH.
2. При входе вы увидите баннер-предупреждение.
3. Введите команду для просмотра живого лога:
   ```bash
   tail -f /var/log/ha_install_reboot.log
   ```

## 🎚 Профили установки

В интерактивном визарде доступно 5 предустановленных профилей и ручной выбор:

| Профиль | Описание | Включено по умолчанию |
| :--- | :--- | :--- |
| **minimal** | Только HA + Docker + OS-Agent. | Базовый набор, без оптимизаций. |
| **standard** | Золотая середина для большинства пользователей. | UFW, Watchdog, Бэкапы, HACS, ZRAM, SSH-защита. |
| **full** | Всё включено. | Всё из `standard` + Tailscale, Cloudflare, Prometheus, удаленные бэкапы. |
| **server** | Для выделенных серверов. | Статический IP, мониторинг, без Cloudflare. |
| **dev** | Для разработчиков. | HA + HACS, без системных твиков. |
| **custom** | Ручной выбор. | Пользователь сам отмечает нужные компоненты. |

## ⚙️ Архитектура и Внутренние механизмы

Скрипт состоит из 14 пошаговых этапов (Steps), каждый из которых можно возобновить при сбое.

### 1. Обход ограничий ОС (`os-release`)
HA Supervised требует `ID=debian`. На Armbian/Ubuntu скрипт:
- Бэкапит оригинальный `/etc/os-release`.
- Создает фейковый `os-release` с `ID=debian` на время установки `.deb` пакета.
- Создает systemd drop-in для `hassio-supervisor.service`, который подменяет файл только на время запуска контейнера Supervisor, оставляя хостовую ОС нетронутой.

### 2. Патчинг загрузчика (AppArmor)
AppArmor должен быть включен в параметрах ядра. Скрипт умеет искать и патчить:
- `armbianEnv.txt` / `uEnv.txt` (ищет `extraargs`, `optargs`, `APPEND` или `rootflags`).
- `extlinux.conf` (ищет строку `APPEND`).
- `cmdline.txt` (Raspberry Pi, дописывает в конец единственной строки).
Аргументы дописываются **строго в продолжение нужной строки**, без переноса на новую, чтобы не сломать загрузчик U-Boot.

### 3. Автоматическое продолжение после перезагрузки
Если для включения AppArmor требуется ребут:
- Скрипт копирует себя в `/usr/local/bin/ha-install`.
- Создает systemd-службу `ha-install-continue.service`.
- Перезагружает систему.
- После загрузки служба запускает скрипт с флагом `--from-step=apparmor`.
- **Защита от цикла:** Счетчик попыток. Если после 3 перезагрузок AppArmor не включается (кривой загрузчик), скрипт продолжает установку без него (HA будет в статусе Unsupported).

### 4. Безопасная настройка сети
Установка NetworkManager "на лету" часто рвет SSH. Скрипт использует хитрость:
- Перед установкой NM создается скрипт `/usr/sbin/policy-rc.d`, который запрещает APT автоматически запускать службы.
- NM устанавливается, но не запускается.
- Скрипт корректно выключает `ifupdown`, настраивает NM и только тогда запускает сеть.
- В случае полного краха сети вызывается `rollback_network`, возвращающий старые конфиги.

### 5. Умное ожидание контейнеров
Шаг `step_install_ha` не просто ставит пакет, а ждет:
- Запуска службы `hassio-supervisor`.
- Поочередного поднятия 5 системных контейнеров (dns, cli, audio, multicast, observer) и самого `homeassistant`.
- Если Supervisor зависает в процессе, скрипт его перезапускает.

## 💻 Командная строка и Флаги

Скрипт поддерживает автоматическую (тихую) установку через флаги.

### Режимы работы:
*   `--check` — Диагностика системы (проверка портов, Docker, AppArmor).
*   `--status` — Живой мониторинг в терминале (RAM, CPU, контейнеры).
*   `--update` — Обновление OS-Agent и HA Supervised до последних версий с GitHub.
*   `--uninstall` — Удаление HA (Standard или Full режим).
*   `--rescue` — Режим спасения. Пытается починить ФС, сеть, Docker и Supervisor.
*   `--benchmark` — Тест железа (CPU, RAM, Диск) с рекомендацией профиля.
*   `--self-update` — Обновление самого скрипта `ha-install`.
*   `--export-config` — Экспорт текущих настроек в скрипт-файл.

### Опции установки:
*   `--profile <ИМЯ>` — Указать профиль (minimal, standard, full, server, dev).
*   `--silent` — Тихий режим (без визарда).
*   `--auto-reboot` — Разрешить скрипту перезагружать систему без спроса (для cron/systemd).
*   `--timezone <ЗОНА>` — Например, `Europe/Moscow`.
*   `--data-dir <ПУТЬ>` — Перенос данных HA и Docker на внешний диск.
*   `--wifi <SSID> <ПАРОЛЬ>` — Настройка Wi-Fi.
*   `--tailscale` / `--ts-authkey <КЛЮЧ>` — Установка Tailscale с авто-авторизацией.
*   `--cloudflared` / `--cf-token <ТОКЕН>` — Настройка Cloudflare Tunnel.

**Пример тихой установки профиля Full с авторебутом:**
```bash
sudo ha-install --profile full --silent --auto-reboot
```

## 🧩 Меню модулей

Если вы установили HA с профилем `minimal`, а позже решили добавить функции, не переустанавливайте систему. Запустите `sudo ha-install` и выберите `Install modules`.

Доступные модули:
- **ZRAM Swap** — Сжатие памяти в RAM.
- **eMMC Tuning** — Оптимизация износа накопителя (noatime, лимиты journald).
- **Watchdog** — Перезапуск HA при зависании с экспоненциальным откатом.
- **Backups** — Локальные бэкапы + снапшоты через HA CLI.
- **Remote Backup** — Отправка бэкапов по SSH (rsync) или в облако (rclone: Yandex, GDrive).
- **Notifications** — Настройка Telegram, ntfy.sh, Discord, Slack.
- **HACS** — Магазин кастомных интеграций.
- **Tailscale / Cloudflare** — VPN и публичный HTTPS доступ.
- **Security** — UFW + изоляция Docker + харденинг SSH.

## 🛠 Утилиты, добавляемые в систему

Скрипт генерирует несколько полезных команд в `/usr/local/bin/`:

- `ha-health` — Выводит сводку: IP, температура, RAM, статус контейнеров и код ответа HA Web (8123).
- `ha-backup` — Умный бэкап. Если есть HA CLI — делает полный снапшот. Если нет — быстрый tar конфигов.
- `ha-restore` — Интерактивное восстановление из снапшота или tar-архива.
- `ha-notify "текст"` — Отправка уведомления (используется скриптом, но можно запускать руками).
- `ha-watchdog` — Проверка пинга и HTTP. Перезапускает контейнер при необходимости.

## 🆘 Решение проблем (FAQ)

**1. В логе установки APT пишет: "Пакетная установка не удалась, поштучно..."**
*Причина:* APT конфликтует (часто из-за NetworkManager или сломанных зависимостей образа).
*Решение:* Скрипт автоматически переходит в поштучный режим и выводит последние 5 строк лога APT. Если и это падает, проверьте `/etc/apt/sources.list` на наличие мертвых репозиториев.

**2. HA показывает статус "Unsupported".**
*Причина:* Supervisor ожидает HAOS. Так как вы на Debian/Armbian, он помечает систему как неподдерживаемую.
*Решение:* Игнорировать. На работу автоматизаций и аддонов это не влияет. Если AppArmor выключен, скрипт также предупредит об этом.

**3. Машина ушла в циклическую перезагрузку на шаге AppArmor.**
*Причина:* Загрузчик U-Boot игнорирует аргументы ядра (крайне редкий случай).
*Решение:* Скрипт имеет защиту — после 3 перезагрузок он прекратит попытки и продолжит установку без AppArmor. Если цикл не прекращается, отформатируйте загрузочный раздел и запишите загрузчик заново.

**4. После ребута не могу понять, работает ли скрипт.**
*Решение:* Зайдите по SSH. Если скрипт работает, вы увидите красный баннер "HA УСТАНОВЩИК ПРОДОЛЖАЕТ РАБОТУ В ФОНЕ". Выполните `tail -f /var/log/ha_install_reboot.log`.

**5. Docker не видит USB-донгл (Zigbee/Z-Wave).**
*Причина:* Установлен тип машины `qemuarm-64`. Для него Supervisor ограничивает проброс USB.
*Решение:* Переустановите HA с явным указанием типа: `sudo ha-install --machine raspberrypi4-64` (если у вас RPi) или проверьте, что в `config` прописан правильный тип.
